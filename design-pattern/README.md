# Getting Started

### 设计模式的作用

* 找出应用中可能需要变化之处，把它们独立出来，不要和那些不需要变化的代码混在一起。
* 针对接口编程而不是针对实现编程
* 多用组合，少用继承
* 为了交互对象之间的松耦合设计而努力

### 继承和接口的弊端

* 需求变更的时候
* 通过修改超类让所有子类加上方法的弊端在并非所有的子类都需要这种拓展
* 通过接口拓展的问题在于接口没法实现，子类无法复用
